User Guide

WordFish is a platform for markup and annotation of text and images.

The standard flow of machine learning research usually looks something like this:



We unfortunately run into bottlenecks. Let's talk about each of the steps, and how WordFish is going to help us.

Step 1: Organize your data

Getting the data, and moving it around is usually hard. Using the WordFish standard, you can zip up your data, and upload it to a new collection, and you are ready to go. What does the standard look like? It's just a simple organization of images and text files in folders on your computer. If you need help validating a dataset, we provide a tool to do that, along with sample datasets.

Step 2: Annotate/Markup Images and Text



Annotation

Annotation means that you want to ascribe a specific label to some text or all or part of an image. Usually, we want to annotate datasets for supervised machine learning, because we want to use features from the text or image to predict the label. In the case of annotation, we want to define a clear set of labels to use, each with one or more options (e.g., label "disease" has options "present" and "absent." WordFish provides an easy way to do this, for either text or images.

  • Define labels for your collection
  • Send out a link to annotate images or text for those labels

Markup

Markup is like freehand drawing. Whether you are looking at text or an image, it means that you are free to circle or highlight an idea to indicate something general like "this is important." An image markup task might ask a radiologist to circle an area of interest, and this area might be used to direct an unsupervised classifier to focus on an area of an image. A text markup task might mean asking a political scientist to highlight the sentence(s) that were most convincing for a report. A markup gives the user freedom to select or highlight information. Using WordFish, after you upload your data, you can do different kinds of markup:

  • Describe means that the user can write openly to describe an image or text. For example, you might ask a user to write about how a painting makes him or her feel.
  • Markup Images means that the user can draw freely on your image. You can provide any (or no) instruction on how you want this done. An overlay is saved with the markup with a matching image, so you can carry the markup and image forward to your machine learning task.
  • Markup Text This comes down to highlighting words or chunks of text. For example, we might ask a user to highlight ingredients in a recipe, and then train a classifier to do the same.

Step 3: Download your data for Machine Learning

Once your data is in WordFish, the annotations, markups, and original images are available via our API for you to use programatically for machine learning. You can also export your dataset from the interface, or link up directly with cloud machine learning services (under development).

Step 4: Discovery

This is up to you, dawg. Once you've made it here, by using WordFish you can be ensured that the entire process, from moving data from it's origin through machine learning, is programmatic and reproducible. When you have more data, it plugs in seamlessly.
